Smart Meal Recommendation Bot - Codebase Guide

This page explains the full app structure in one place so a teammate can read it first, understand the flow, and then open the right file when they need to change something.

Project Purpose

The app is a Streamlit meal recommendation bot for SIT788 Option 2. It accepts typed preferences, an uploaded food or ingredient image, a short voice preference, or any combination of those inputs. It uses Azure AI Vision for image understanding, ingredient filtering for specific food terms, Azure Speech to Text for voice transcription, and a local CSV recipe dataset for recommendation scoring.

Text input Image input Voice input Azure AI Vision Azure Speech Local recipe CSV

How The App Runs

1. app.py Starts Streamlit, loads config status, renders sidebar, and loads recipes.
2. Components Draw text input, image upload, voice input, combined profile, and recommendations.
3. Azure Services Analyse the uploaded image and transcribe recorded speech when credentials exist.
4. Search Profile Combine whichever sources exist: typed text, specific image ingredients, OCR text, and voice transcript.
5. Recommender Tokenize the search profile, match the main protein, score recipes, and show ranked meals.

Main Files

File What It Does When To Edit It

Components

File Functions Responsibility
components/input_section.py render_text_input, render_image_input, render_voice_input Builds the three user input areas: typed preference, image upload, and voice recording or WAV upload fallback.
components/sidebar.py render_sidebar Shows system status, including whether the local dataset is ready and whether Azure Vision and Speech credentials are configured.
components/results.py render_combined_text, render_recommendations Shows the uploaded image used for the result, the combined search profile, and ranked recipe recommendations with score, ingredients, tags, protein match, and matched terms.

Config

File Functions Responsibility
config/settings.py get_config_status, get_required_env, AzureConfigStatus Loads the .env file, checks Azure credentials, and gives the app a simple ready or missing status.

Services

File Functions Responsibility
services/azure_vision.py analyse_image Calls Azure AI Vision to extract image caption, tags, detected objects, and OCR text from the uploaded image.
services/ingredient_detector.py detect_image_ingredients, enrich_image_analysis_with_ingredients Filters raw Azure Vision output against the recipe ingredient vocabulary so image recommendations use specific ingredients instead of broad labels.
services/azure_speech.py transcribe_audio Saves the recorded audio temporarily, sends it to Azure Speech to Text, and returns the transcript.
services/data_loader.py load_recipes Loads data/recipes.csv, validates required columns, and caches the dataset for Streamlit.
services/text_processing.py tokenize Cleans text into useful keywords by lowercasing, removing small words, and ignoring common stop words.
services/recommender.py recommend_recipes Scores each recipe by matching query keywords against recipe name, ingredients, tags, cuisine, and meal type.

Dataset

The local recipe data lives in data/recipes.csv. Each recipe must include these columns: 

name,cuisine,ingredients,tags,prep_time,meal_type

The recommender uses these fields for scoring. For example, a user search profile containing quick high protein chicken rice dinner strongly matches recipes that include those words in ingredients and tags. If the input contains a clear protein such as beef, pork, chicken, fish, seafood, tofu, egg, or turkey, the recommender keeps recipes aligned with that protein and filters incompatible meals.

Recommendation Scoring

The recommender first checks for a main protein. This prevents a beef image or beef text request from returning unrelated fish or sushi recipes. Compatible fish and seafood terms can still match each other, but exact protein matches receive the strongest bonus.

Recipe Field Weight Why
main protein 10 bonus Exact protein matches such as beef, pork, chicken, fish, seafood, tofu, egg, or turkey are strongly preferred.
name 4 The recipe name is highly important because it usually contains the core meal.
ingredients 3 Ingredients directly match what the user has or wants.
tags 3 Tags capture preferences such as healthy, quick, vegan, spicy, or high protein.
cuisine 2 Cuisine helps when the user asks for Italian, Indian, Thai, and similar terms.
meal_type 2 Meal type helps match breakfast, lunch, or dinner requests.

Data Flow In Plain English

  1. The user types a preference, uploads an image, and records or uploads audio.
  2. The user can also use only one of those inputs, such as image only or speech only.
  3. app.py sends the image bytes to analyse_image.
  4. app.py sends the audio bytes to transcribe_audio.
  5. The app filters raw image caption, tags, and objects into specific recipe ingredients.
  6. The app combines the available typed text, specific image ingredients, OCR text, and voice transcript.
  7. recommend_recipes tokenizes that combined text and detects the requested main protein.
  8. Recipes with incompatible proteins are filtered when a clear protein is detected.
  9. The highest scoring recipes are displayed with the uploaded image and matched terms so the result is explainable.

Where To Change Things

Change UI wording

Edit components/input_section.py, components/results.py, or components/sidebar.py.

Add more recipes

Edit data/recipes.csv and keep the same column names.

Change scoring rules

Edit FIELD_WEIGHTS or matching logic in services/recommender.py.

Change text cleanup

Edit stop words or token rules in services/text_processing.py.

Change Azure Vision behavior

Edit visual features or result parsing in services/azure_vision.py.

Change Azure Speech behavior

Edit language, temp audio handling, or result handling in services/azure_speech.py.

Setup Reminder

Copy .env.example to .env, add Azure keys, install dependencies with pip install -r requirements.txt, then run streamlit run app.py.
